1) Введение.


  Система Webroute была спроектирована и разработана для осуществления контроля доступа пользователей к сети Интернет , предоставления авторизированным пользователям доступа к локальной сети предприятия, посредством технологии виртуальных частных сетей а также для защиты локальной сети от атак из Интернет.
Система позволяет:
  • Надежно авторизировать пользователей при доступе к сети Интернет.
  • Управлять пользователями системы (добавлять, удалять, менять пароль).
  • Предоставлять доступ из вне к сети предприятия с использованием шифрования передоваемых данных и аутентификацией пользователей с использованием сертификатов(ключей).
  • Контролировать доступ к сети Интернет как с помощью предопределенных политик, так и с помощью своих собственных.
  • Контролировать пропускную способность и скорость доступа определенных пользователей к сети Интернет.
  • Назначать как дневные так и месячные квоты трафика для конкретных пользователей системы.
  • Формировать отчеты по посещению пользователями web страниц и количеству потребленного ими трафика.
  • Просматривать информацию о состоянии аппаратной части системы.
Для доступа к web интерфейсу необходимо набрать в адресной строке вашего браузера: http://gate или http://gate.loc

2) Общее устройство системы.


  Логически всю систему можно разделить на несколько подсистем:
1) Web интерфейс;
2) Система авторизации;
3) Система генерации отчетов;
4) Система управления трафиком;
5) Система контроля доступа к Web;

Каждый из этих модулей работает самостоятельно и обращается к другим модулям через специальные интерфейсы.
Этим объясняются некоторые особенности работы системы о которых речь пойдет далее.
Для повседневного использования системы и упрощения администрирования был разаработан web интерфейс.
Для доступа к web интерфейсу необходимо набрать в адресной строке вашего браузера: http://gate или http://gate.loc
Вначале откроется станица авторизации web интерфейса. На ней следует ввести логин и пароль администратора системы.
По умолчанию логин: admin пароль: admin . (Внимание обязательно смените пароль, о том как это сделать будет расказано далее).
После входа в систему Вы получите доступ к настройкам системы и отчетам.

3) Пользователи.

Сам по себе web интерфейс разделен на две составляющих: это горизонтальное меню и один из модулей настроек, которые можно выбрать при помощи меню.
Меню состоит из двух уровней: первый - это категория(модуль меню), которая объединяет пункты меню, а второй уровень - это непосрественно сам пункт этого меню.

Первая категория это пользователи. Содержит в себе следущие пункты:

  a) Список пользователей. - Простой список пользователей, их паролей и прав.
  b) Добавить нового. С помощью этого пункта меню Вы можете добавлять новых пользоватей в систему. Для этого необходимо в соответсвующие поля ввести логин и пароль создаваемого пользователя. Логин и пароль могут состоять только из цифр латинского алфавита и/или цифр. Также есть возможность выбрать роль нового пользователя в системе(по-умолчнию ему будет присвоена роль "пользователь").
Администратор - имеет полный и ничем не ограниченный доступ к системе.
Просмотр статистики - имеет только возможность просматривать статистику.
Пользователь - не имеет ни каких прав доступа к web интерфейсу.
  c) Удалить пользователя. Навсегда удаляет пользователя из системы.
  d) Изменить пароль. Предоставляет возможность изменения пароля любого пользователя в системе.

4) Контроль доступа к web.

Вторая категория меню, служит для ограничения возможности посещения определенных сайтов.
Все сайты представленны в виде категорий. (например порно или сайты знакомств или социальные сети).
a) Список категорий. Отображает информацию обо всех существующих категориях и позволяет включать или выключать отдельные категории.
b) Абсолютные исключения. Список пользователей для которых контроль доступа к web не осуществляется - можно посейщать любые сайты(но статистика все равно ведется).
c) Исключения по категориям. Список пользователей для которых не действуют какие-то конкретные категории(например : для Васи не работает категория социальные сети - значит что Вася может посещать сайты относящиеся к этой категории).
d) Исключения по пользователям. Список пользователей для которых не действуют какие-то конкретные ограничения (например : для Васи работает категория социальные сети, но в исключениях есть vkontakte.ru - значит что Вася не может посещать сайты относящиеся к этой категории, но может зайти на сайт vkontakte.ru).
e) Запрещения по пользователям. Список пользователей для которых действуют какие-то конкретные ограничения (например : для Васи запрещен сайт mail.ru - значит что Вася (и только Вася) не может посещать mail.ru).
f) Разрешенные сайты. Список сайтов которые можно посейщать всем пользователям без исключения (данная категория служит для того чтобы разрешить сайт который оказался в какой либо из автоматически обновляемых категорий).
g) Запрещенные сайты. Список сайтов которые Вы хотите запретить, если он вдруг не оказался в какой либо из автоматически обновляемых категорий.
h) Страница блокировки. Настройки относящиеся к странице, которую увидит пользователь если попытается зайти на заблокированный сайт.
1. Показывать форму заявки на разблокировку сайта? -- Показывать ли форму через которую пользователи могут сообщить об ошибочно заблокированном сайте или просто поросить администратора разблокировать им этот сайт.
2. Адрес, куда будут отправляться заявки. -- Адрес электронной почты куда следует отправлять заявки.
3. Требовать от пользователя обратный e-mail? -- Обязательно ли для пользователя указывать адрес электронной почты, по которому с ним (с пользователем) можно связаться.

5) Управление трафиком.

Служит для управления скоростью пользвательских подключений а также назначения квот на количество трафика.
a) Ограничение скорости. Служит для ограничения скорости пользовательского соединения с Интернет. По умолчанию у пользователей скорость никак не ограничена.
b) Назначение квот. Служит для назначения квот на трафик. Если какое либо из значений равно нулю то это значение не учитывается. Также если для одного пользователя имеется более одной квоты, то их значения складываются (удобно использовать для временного увеличения квоты). Если квота была превышена и пользователь был заблокирован то в графе причина будет указано что именно он превысил. Так month_in - месячная квота входящего трафика, month_out - месячная квота исходящего трафика, day_in - дневная квота входящего трафика, day_out - дневная квота исходящего трафика.
c) Протоколы L7. Служит для включения/выключения анализа трафика по различным протоколам. Внимание, чем больше включено протоколов, тем больше нагрузка на сервер(на процеесор сервера). Также обратите внимание на параметр скорость (от 1 до 4) чем он меньше, тем сильнее нагрузка.
d) Фильтрация L7. Содержит форму для добавления фильтров, а также таблицу уже созданных правил(с помощью таблицы можно и удалять правила).

6) Отчеты.

Служит для просмотра сформированных отчетов по трафику и посещению web страниц.
Все отчеты генерируются автоматически. Из-за довольно большой нагрузки на систему генерация отчетов происходит раз в сутки(ночью), по этой причине все отчеты можно просмотреть только за прошедшие дни.
Отчеты делятся на два типа:
a) Отчет по трафику. Общий отчет по трафику израсходованому пользователями за прошедшие дни, месяцы или года.
b) Отчет посещения WEB. Детализированый отчет по посещению web страниц пользователями системы.
c) Настройка отчетов. Позволяет исключить выбранного пользователя из будущих отчетов WEB статистики.
(Также по запросу возможно исключить пользователя и из общей статистики по трафику).
